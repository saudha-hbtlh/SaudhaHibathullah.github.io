🚲 CitiBike Flow – Demand Alert & Forecasting System

BigQuery ML · ARIMA_PLUS · Looker Studio · SQL

🎬 Watch CitiBike Flow in action – forecast demand, detect imbalances, and optimize bike distribution

🛠️ Tools & technologies:

Google Cloud Platform BigQuery & BigQuery ML ARIMA_PLUS Looker Studio SQL & Views Python (helper scripts)

Situation:

Bike-sharing systems like Citi Bike face a daily operational challenge: stations either run out of bikes or become completely full, frustrating users and reducing system efficiency. With over 550 stations and thousands of trips per day, manually monitoring demand is impossible. Without a data-driven solution, the system risks losing users to unreliable service.

Task:

Build an end-to-end automated forecasting and alerting system to:
✅ Predict daily trip demand for every station 30 days in advance
✅ Compare forecasted demand against station capacity
✅ Detect supply-demand mismatches before they happen
✅ Visualize alerts in an interactive dashboard
✅ Scale to hundreds of stations with zero manual effort

Action:

The system was built entirely on the Google Cloud Platform, using the public Citi Bike trip dataset from 2014-2015, and consists of four core components:

📊 Data Processing: The raw trip data was aggregated into daily time series for each station, providing clean input for the forecasting model.

🧠 Forecasting Model (ARIMA_PLUS): A multi-time-series forecasting model was trained using BigQuery ML's ARIMA_PLUS algorithm. The model automatically detects trends, weekly seasonality, yearly patterns, and holiday effects, generating unique forecasts for all 550+ stations simultaneously with a single SQL query. A `forecast_limit_lower_bound = 0` was applied to prevent negative trip predictions.

⚖️ Capacity Logic: Station capacity was defined using the 90th percentile of historical daily trips, meaning that for 90% of days the station's demand stays at or below this number. This creates a robust, realistic baseline for alerting.

🚨 Alert System: A SQL view compares the average daily forecasted demand against station capacity for a 7-day window. Stations are categorized into five alert levels based on the demand gap: Critical Overflow (demand exceeds 120% of capacity), Warning Overflow, Balanced, Warning Empty, and Critical Empty (demand below 30% of capacity).

Result:

✅ Forecasts generated for 550+ stations with a single query — enterprise-grade scalability
✅ 327 alerts identified for the March 6–12, 2015 forecast window
✅ Interactive Looker Studio dashboard with:
   📌 Executive Summary with KPI scorecards & critical alerts table
   📌 Station‑level demand forecast with confidence bands
   📌 Filterable station status table with conditional formatting
✅ Average forecast confidence of 95% – reliable predictions
✅ Operations teams can now focus on 5% of stations (critical alerts) instead of monitoring all 550+
